## 4.2 Web安全扫描模块实现（完善版）

Web 安全扫描能力由 `src/cores/web/`（底层抓取/解析/爬虫）与 `src/modules/web_scan/`（目录扫描、Fuzz、子域名枚举封装）共同实现。整体采用“核心能力 + 模块化编排”的分层设计：

- `cores/web/fetcher.rs`：并发 HTTP 抓取、重试、退避、Host 级并发限制。
- `modules/web_scan/dir_scan.rs`：目录爆破、结果过滤、近似去重、断点续扫。
- `modules/web_scan/fuzz_scan.rs`：`FUZZ` 占位符替换、批量并发请求与异常结果筛选。
- `modules/web_scan/dns_scan.rs`：基于 DNS 解析 + HTTP/HTTPS 可达性验证的子域名发现。
- `cores/web/parser.rs` + `cores/web/crawl.rs`：HTML 链接提取与并发爬虫调度。

### 4.2.1 并发 HTTP 请求引擎实现

本系统基于 `reqwest::Client` 构建异步请求引擎，通过 `fetch_many(...).buffer_unordered(concurrency)` 提供全局并发能力，并使用“每主机信号量”实现 host 级并发隔离。

```rust
// src/cores/web/fetcher.rs
pub async fn fetch_many<I>(
    &self,
    reqs: I,
    concurrency: usize,
) -> Vec<Result<FetchResponse, RustpenError>>
where
    I: IntoIterator<Item = FetchRequest>,
{
    stream::iter(reqs)
        .map(|r| {
            let this = self.clone();
            async move { this.fetch_with_request(r).await }
        })
        .buffer_unordered(concurrency)
        .collect()
        .await
}
```

Host 级并发控制通过 `HashMap<String, Arc<Semaphore>>` 动态维护，每个 host 独立限流：

```rust
// src/cores/web/fetcher.rs
let sem_arc = if let Some(s) = self.per_host_semaphores.read().await.get(&host).cloned() {
    s
} else {
    let mut map = self.per_host_semaphores.write().await;
    map.entry(host.clone())
        .or_insert_with(|| Arc::new(Semaphore::new(self.config.per_host_concurrency)))
        .clone()
};
let _permit = sem_arc.acquire_owned().await?;
```

错误恢复方面，实现了 `429/5xx/网络错误` 重试与指数退避，并支持优先遵循 `Retry-After` 头：

```rust
// src/cores/web/fetcher.rs
if resp.status().as_u16() == 429 {
    if self.config.honor_retry_after
        && let Some(ra) = resp.headers().get("retry-after")
        && let Ok(s) = ra.to_str()
        && let Some(dur) = Self::parse_retry_after_seconds_or_date(s)
    {
        tokio::time::sleep(dur).await;
        continue;
    }
    let backoff = Self::exponential_backoff_ms(
        self.config.backoff_base_ms,
        self.config.backoff_max_ms,
        attempts,
    );
    tokio::time::sleep(Duration::from_millis(backoff)).await;
    continue;
}
```

说明：当前版本未引入 `governor` 令牌桶，也未显式设置 `pool_max_idle_per_host/pool_idle_timeout`；连接复用由 `reqwest::Client` 默认连接池机制完成。

### 4.2.2 目录爆破算法实现

目录扫描入口为 `run_dir_scan`。系统先构造请求集合，再并发发送并执行状态码过滤、wildcard 过滤和指纹去重。

```rust
// src/modules/web_scan/dir_scan.rs
for p in &effective_paths {
    let url = build_joined_url(base, p);
    if !seen_req.insert(url.clone()) {
        continue;
    }
    reqs.push(FetchRequest {
        url,
        method: cfg.request_method.clone(),
        headers: cfg.request_headers.clone(),
        body: render_request_body(&cfg.request_body_template, None),
        timeout_ms: cfg.timeout_ms,
        max_retries: cfg.max_retries,
        follow_redirects: Some(cfg.follow_redirects),
    });
}
```

URL 拼接对前后斜杠进行规范化处理：

```rust
// src/modules/web_scan/common.rs
pub fn build_joined_url(base: &str, path: &str) -> String {
    if path.starts_with('/') {
        format!("{}{}", base.trim_end_matches('/'), path)
    } else {
        format!("{}/{}", base.trim_end_matches('/'), path)
    }
}
```

目录扫描支持两级降噪：

1. wildcard 签名过滤：先请求随机探针路径，形成“默认 404 页/兜底页”签名。
2. 指纹近似去重：基于 `title + simhash` 的汉明距离判定相似页面。

```rust
// src/modules/web_scan/common.rs
let fp = build_fingerprint(resp.status, &resp.body);
if is_near_duplicate(&fp, &seen_fingerprints, cfg.fingerprint_distance_threshold) {
    continue;
}
seen_fingerprints.push(fp);
```

此外，目录扫描实现了 `resume_file` 断点续扫，可在大字典任务中断后恢复进度。

### 4.2.3 Web Fuzz 测试实现

Fuzz 模块采用模板化 URL（必须包含 `FUZZ` 占位符），将关键词逐个替换并并发请求：

```rust
// src/modules/web_scan/fuzz_scan.rs
if !url_with_fuzz.contains("FUZZ") {
    return Err(RustpenError::ParseError(
        "fuzz url must include FUZZ placeholder".to_string(),
    ));
}
for kw in keywords {
    let url = url_with_fuzz.replace("FUZZ", kw);
    reqs.push(FetchRequest {
        url,
        method: cfg.request_method.clone(),
        headers: cfg.request_headers.clone(),
        body: render_request_body(&cfg.request_body_template, Some(kw)),
        timeout_ms: cfg.timeout_ms,
        max_retries: cfg.max_retries,
        follow_redirects: Some(cfg.follow_redirects),
    });
}
```

结果分析维度包括：

- 状态码范围过滤（`status_min/status_max`）
- 响应长度过滤（`content_len_min/content_len_max`）
- wildcard/指纹去重过滤

为降低高压目标上的误判与超时，模块提供了基于 `429/5xx` 比例的自适应延迟策略：

```rust
// src/modules/web_scan/fuzz_scan.rs
if cfg.adaptive_rate && observed > 0 {
    if throttle_hits * 3 >= observed {
        adaptive_delay_ms = (adaptive_delay_ms.saturating_mul(2).saturating_add(20))
            .min(cfg.adaptive_max_delay_ms);
    } else {
        adaptive_delay_ms /= 2;
    }
}
```

### 4.2.4 DNS 子域名枚举实现

当前版本实现的子域名发现流程为“字典生成候选 + DNS 解析验证 + HTTP/HTTPS 可达性探测”。

1. 字典拼接：`{word}.{root_domain}`。
2. DNS 验证：`tokio::net::lookup_host` 判断可解析性。
3. 服务验证：默认开启 `dns_http_verify`，对候选域名发起 HTTP，失败时回退 HTTPS。

```rust
// src/modules/web_scan/dns_scan.rs
match tokio::net::lookup_host(format!("{fqdn}:{resolve_port}")).await {
    Ok(mut iter) if iter.next().is_some() => Some(ProbeCandidate::Remote { ... }),
    _ => None,
}
```

```rust
// src/modules/web_scan/dns_scan.rs
let req_http = build_dns_http_request("http", &fqdn, port, &word, cfg);
match fetcher.fetch_with_request(req_http).await {
    Ok(resp) if status_in_scope(resp.status, cfg) => Ok(Some(to_module_result(resp))),
    _ => {
        let req_https = build_dns_http_request("https", &fqdn, port, &word, cfg);
        // HTTPS fallback
        ...
    }
}
```

该模块还支持本地靶场/CTF 场景：当目标是 `localhost` 或 IP 时，通过 `Host` 头模拟虚拟主机枚举。

说明：当前代码尚未实现 CT 日志抓取、搜索引擎枚举与 AXFR 区传输；论文中应将其定义为“可扩展枚举策略”，而非“已实现能力”。

### 4.2.5 HTML 解析与链接提取

HTML 解析器位于 `parser.rs`，当前实现聚焦 `<a href>` 链接提取，并将相对路径解析为绝对 URL：

```rust
// src/cores/web/parser.rs
let selector = Selector::parse("a[href]")?;
for el in document.select(&selector) {
    if let Some(href) = el.value().attr("href")
        && let Ok(resolved) = base.join(href)
    {
        links.push(resolved.to_string())
    }
}
```

爬虫采用并发 BFS，支持深度、并发数、最大页面数约束，并缓存 `robots.txt` 策略：

```rust
// src/cores/web/crawl.rs
pub struct CrawlConfig {
    pub max_depth: usize,
    pub concurrency: usize,
    pub max_pages: Option<usize>,
    pub obey_robots: bool,
    pub default_host_delay_ms: u64,
}
```

`robots.txt` 解析支持 `User-agent: *` 下的 `Disallow` 与 `Crawl-delay`，并在请求前执行每主机访问间隔控制（politeness）。

说明：URL 去重当前基于 `HashSet<String>`，尚未采用 Bloom Filter。若后续面向千万级 URL，可将 Bloom Filter 作为内存优化改造方向。

### 4.2.6 小结与论文表述修订建议

结合当前代码，建议论文 4.2 节采用“已实现能力 + 可扩展方向”双层表述：

- 已实现：并发抓取、Host 级并发控制、指数退避、目录/Fuzz wildcard + simhash 降噪、DNS 解析与 HTTP 验证、robots 约束爬取。
- 可扩展：`governor` 令牌桶全局速率限制、CT/搜索引擎/AXFR 多源子域名发现、Bloom Filter URL 去重、多标签资源提取（`img/script/link`）。

该写法可以确保论文描述与代码一致，降低答辩中“描述超前于实现”的风险。
