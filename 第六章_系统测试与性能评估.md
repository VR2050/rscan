## 6.3 对比分析

### 6.3.1 与传统工具功能对比

按仓库功能清单可确认：

1. `rscan` 单二进制覆盖 `host + web + vuln + reverse + TUI/Zellij`。
2. `ffuf/gobuster/nmap/rustscan` 通常聚焦单域能力。
3. reverse 作业管理与统一 TUI 工作台是本项目差异化能力。

[图6-10 占位：功能矩阵对比图]

### 6.3.2 性能优势分析

证据来源分两类：

1. 可复现脚本（Web 对比与门禁）。
2. 文档历史样例（Host 小基准）。

结合 `6.2.5` 集中实测区，可得工程结论：

1. 项目在实网靶机上可稳定输出结构化结果与可用耗时数据。
2. Web 侧具备与主流工具做持续门禁对比的能力。
3. Host 侧当前仍缺自动化对标脚本的长期归档，后续可补齐。

[图6-11 占位：性能分析图（脚本基线 + 在线实测）]

### 6.3.3 用户体验对比

从实现可确认：

1. 统一参数体系降低多工具切换成本。
2. TUI + Zellij managed tabs 提升任务可视化与原生终端协同。
3. reverse 工作区支持文件选择、任务回流、日志/产物联动。

当前仓库无问卷类主观满意度数据，故只给工程侧体验结论，不给主观评分统计。
