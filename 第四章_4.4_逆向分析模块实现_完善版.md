    ## 4.4 逆向分析模块实现（完善版，侧重 Rust 实现）

    逆向分析模块位于 `src/modules/reverse/`，采用“Rust 原生分析 + 外部后端编排 + 作业化管理”架构，核心能力包括：

    - 静态样本分析（ELF/PE/APK）：`analyzer.rs`
    - 受管反编译作业（job 生命周期、日志、产物、健康检查）：`jobs.rs`
    - Ghidra/JADX/radare2 后端编排：`backends/*` + `orchestrator.rs`
    - IR 统一抽象与适配：`ir.rs` + `adapters/*`
    - 交互式检索与 Tantivy 索引：`console.rs`

    ### 4.4.1 Ghidra 集成适配器实现

    项目通过后端注册机制管理 Ghidra Headless，可从环境变量、内置目录和系统 PATH 自动探测可执行入口。

    ```rust
    // src/modules/reverse/backends/mod.rs
    if let Ok(home) = env::var("RSCAN_GHIDRA_HOME") {
        let base = expand_tilde(&home);
        let run = base.join("run-headless.sh");
        let bin = base.join("support").join("analyzeHeadless");
        ...
    }
    ```

    同时支持内置精简版路径探测（与论文中的 `third_party/ghidra_core_headless_x86_min/` 对应）：

    ```rust
    // src/modules/reverse/backends/mod.rs
    p.join("third_party/ghidra_core_headless_x86_min")
    ```

    Ghidra 执行命令由 Rust 侧动态构造，统一使用 `analyzeHeadless`：

    ```rust
    // src/modules/reverse/tooling.rs
    ToolInvocation {
        program: "analyzeHeadless".to_string(),
        args: vec![
            out.display().to_string(),
            "rscan_project".to_string(),
            "-import".to_string(),
            input.display().to_string(),
            "-scriptPath".to_string(),
            out.display().to_string(),
            "-postScript".to_string(),
            "ghidra_export_pseudocode.java".to_string(),
            "pseudocode.jsonl".to_string(),
        ],
        ...
    }
    ```

    Ghidra Java 脚本不是手工维护文件，而是由 Rust 代码生成并写出（伪代码/索引/单函数三种脚本），输出 JSONL。

    ```rust
    // src/modules/reverse/backends/ghidra.rs
    match mode {
        DecompileMode::Full => ReverseTooling::write_ghidra_export_script(&script)?,
        DecompileMode::Index => ReverseTooling::write_ghidra_index_script(&script)?,
        DecompileMode::Function => ReverseTooling::write_ghidra_function_script(&script)?,
    }
    ```

    缓存与复用方面，当前实现采用“工程级缓存复用”策略（非简单文件级缓存）：

    - `RSCAN_GHIDRA_PROJECT_CACHE`：控制是否启用工程缓存
    - `RSCAN_GHIDRA_REUSE_PROJECT`：控制是否复用已存在工程
    - 项目名由输入路径 + 元信息哈希生成，降低重复导入成本

    ### 4.4.2 多格式二进制文件支持

    模块支持 `ELF/PE/APK` 三类格式识别。识别首先基于魔数和关键结构，再进入格式解析流程。

    ```rust
    // src/modules/reverse/analyzer.rs
    pub fn detect_format(bytes: &[u8]) -> BinaryFormat {
        if bytes[0..4] == [0x7f, b'E', b'L', b'F'] { ... }
        if &bytes[0..2] == b"MZ" { ... PE\0\0 ... }
        if &bytes[0..4] == b"PK\x03\x04" { ... AndroidManifest/classes.dex ... }
    }
    ```

    ELF/PE 的头部与符号解析基于 `goblin`：

    ```rust
    // src/modules/reverse/analyzer.rs
    match Object::parse(&bytes) {
        Ok(Object::Elf(elf)) => { ... }
        Ok(Object::PE(pe)) => { ... }
        ...
    }
    ```

    APK 分析采用 ZIP 结构遍历，提取 `AndroidManifest.xml`、`classes*.dex`、`lib/*.so` 等关键信息；APK 深度源码反编译则通过 `jadx` 后端作业实现。

    ### 4.4.3 静态分析与反编译流程

    当前实现是“Rust 原生静态分析 + 可选后端深度反编译”的双路径：

    1. `reverse analyze`：快速结构与安全信号分析（导入、节区、熵、可疑字符串等）。
    2. `decompile-run`：受管作业输出 `index/function/full` 工件。
    3. `Rust-first` 模式可直接产出轻量索引与函数级调用/CFG/汇编预览。

    Rust 原生反汇编链路在 `jobs.rs` 中实现：

    - x86/x64：`iced_x86`
    - ARM/ARM64：`capstone`

    ```rust
    // src/modules/reverse/jobs.rs
    let mut decoder = Decoder::with_ip(
        bitness,
        &bytes[region.file_start..end],
        region.va_start,
        DecoderOptions::NONE,
    );
    ```

    ```rust
    // src/modules/reverse/jobs.rs
    let cs = Capstone::new()
        .arm64()
        .mode(capstone::arch::arm64::ArchMode::Arm)
        .detail(false)
        .build()?;
    ```

    在反汇编过程中，系统会构建并输出：

    - 调用边：`calls_functions.jsonl`
    - 交叉引用：`xrefs_functions.jsonl`
    - 控制流块：`cfg_functions.jsonl`
    - 函数汇编：`asm_functions.jsonl`

    这使“控制流图（CFG）/调用关系/局部语义”可被后续检索与展示逻辑复用。

    ### 4.4.4 Tantivy 代码索引与搜索

    Tantivy 索引功能已集成在逆向控制台（`console.rs`）中，索引目录位于 `reverse_out/tantivy`。

    ```rust
    // src/modules/reverse/console.rs
    let mut builder = Schema::builder();
    builder.add_text_field("job_id", STORED);
    builder.add_text_field("ea", STORED);
    builder.add_text_field("name", TEXT | STORED);
    builder.add_text_field("signature", TEXT | STORED);
    builder.add_text_field("all", TEXT);
    ```

    构建索引时，将函数级关键字段写入倒排索引：

    ```rust
    // src/modules/reverse/console.rs
    let _ = writer.add_document(tantivy::doc!(
        job_id_f => job_id,
        ea_f => ea,
        name_f => name,
        sig_f => sig,
        all_f => all
    ));
    ```

    查询使用 `QueryParser + TopDocs`：

    ```rust
    // src/modules/reverse/console.rs
    let qp = tantivy::query::QueryParser::for_index(&index, vec![all_f]);
    let top = searcher.search(&q, &tantivy::collector::TopDocs::with_limit(16))?;
    ```

    说明：该索引主要用于“函数定位与跳转”，并非完整的语义代码搜索系统；但已满足大样本下的快速命中需求。

    ### 4.4.5 轻量动态分析实现

    动态分析通过 `reverse analyze --dynamic` 启用，CLI 会将参数映射为环境变量：

    ```rust
    // src/cli/reverse_exec.rs
    std::env::set_var("RSCAN_REVERSE_DYNAMIC", "1");
    std::env::set_var("RSCAN_REVERSE_DYNAMIC_TIMEOUT_MS", v.to_string());
    std::env::set_var("RSCAN_REVERSE_DYNAMIC_SYSCALLS", v);
    ```

    Linux 下由 `strace` 执行受限追踪（时间窗口 + 指定 syscall 列表）：

    ```rust
    // src/modules/reverse/analyzer.rs
    let mut cmd = Command::new("strace");
    cmd.arg("-f")
    .arg("-qq")
    .arg("-e")
    .arg(format!("trace={}", trace_list))
    .arg("-o")
    .arg(&tmp)
    .arg(path);
    ```

    实现上还包含：

    - 超时终止（防止长时间阻塞）
    - blocklist 关键字预检查（命中则跳过动态执行）
    - 结果提取（`ptrace/prctl/seccomp` 等信号）

    因此该能力更准确地说是“轻量行为信号采样”，而非全量系统调用审计框架。
