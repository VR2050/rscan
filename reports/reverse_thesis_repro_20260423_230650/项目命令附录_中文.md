## A.2.2 host tcp

```text
TCP port scan

Usage: rscan host tcp [OPTIONS] --host <HOST> --ports <PORTS>

Options:
  -H, --host <HOST>
          target host (ip or hostname)

  -l, --log-level <LOG_LEVEL>
          global log level (trace, debug, info, warn, error)
          
          [default: info]

  -p, --ports <PORTS>
          ports to scan (comma separated or repeated)

      --task-workspace <TASK_WORKSPACE>
          Optional workspace for task/TUI integration (writes tasks/<id>/)

  -o, --output <OUTPUT>
          output format: raw/json
          
          [default: raw]

      --task-id <TASK_ID>
          Optional task id (default auto)

  -f, --out <OUT>
          write output to file

      --task-note <TASK_NOTE>
          Optional note saved into task meta

  -s, --service-detect
          Enable service fingerprint enrichment on responses

  -P, --probes-file <PROBES_FILE>
          optional nmap-service-probes path for --service-detect (auto-discovered by default)

      --profile <PROFILE>
          scan profile tuning: low-noise|balanced|aggressive
          
          [default: balanced]
          [possible values: low-noise, balanced, aggressive]

      --tcp-timeout-ms <TCP_TIMEOUT_MS>
          override TCP connect timeout in milliseconds (non-service mode)

      --tcp-concurrency <TCP_CONCURRENCY>
          override TCP scanner concurrency (non-service mode)

      --tcp-retries <TCP_RETRIES>
          override retry count for timeout/filtered ports (non-service mode)

      --tcp-max-rate <TCP_MAX_RATE>
          optional global scan rate limit (ports per second, non-service mode)

      --tcp-jitter-ms <TCP_JITTER_MS>
          optional pacing jitter in milliseconds (non-service mode)

      --tcp-scan-order <TCP_SCAN_ORDER>
          port scheduling strategy: serial|random|interleave (non-service mode)
          
          [possible values: serial, random, interleave]

      --tcp-adaptive-backpressure
          enable adaptive backpressure when filtered/timeout rises (non-service mode)

      --tcp-auto-tune
          auto-tune TCP pacing/concurrency on a sample set before full scan (non-service mode)

      --tcp-mode <TCP_MODE>
          TCP strategy mode: standard|turbo|turbo-verify (non-service mode)

          Possible values:
          - standard:       single pass TCP connect scan
          - turbo:          faster first-pass only, may leave more filtered ports
          - turbo-verify:   turbo first-pass + targeted second-pass verification on filtered ports
          - turbo-adaptive: turbo with adaptive verification strategy (speed/accuracy balance)
          
          [default: standard]

  -h, --help
          Print help (see a summary with '-h')
```
